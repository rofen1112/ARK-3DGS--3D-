# ARK Gaussian Data Contract

Status: Phase 1 active. The browser now consumes the bundled scene data report for default camera fitting.

This document defines the first internal data contract for ARK-3DGS Gaussian scenes. The goal is to let ARK understand the source data before the first-party renderer is built.

## Required PLY Fields

The parser currently requires:

```text
x, y, z
f_dc_0, f_dc_1, f_dc_2
opacity
scale_0, scale_1, scale_2
rot_0, rot_1, rot_2, rot_3
```

Optional SH fields:

```text
f_rest_0 ... f_rest_N
```

## Internal Encoding

ARK records the raw training/export encoding explicitly:

| Field | ARK Encoding | Notes |
|---|---|---|
| center | `float32_xyz` | 3D Gaussian center |
| colorDc | `spherical_harmonics_dc_rgb` | SH DC coefficients, not final sRGB |
| opacity | `logit` | Renderer must apply sigmoid |
| scale | `log_xyz` | Renderer must apply exp |
| rotation | `quaternion_wxyz` | Standard GraphDECO-style PLY order |
| shRest | `spherical_harmonics_rest_rgb` | Higher-order SH terms |

## First-Party Renderer Preview Baseline

ARK now includes a first-party diagnostic WebGL backend:

```text

```

This backend is intentionally limited: it renders decoded PLY centers as SH0-colored circular scale-aware points with CPU back-to-front sorting. It proves ARK-owned parsing, invalid-position filtering, decoded scale/opacity packing, GPU buffer upload, camera fit, canvas rendering, depth-order diagnostics, and visual QA without Aholo.

## Invalid Splat Policy

`decodeGaussianPly()` now has an explicit invalid position policy:

| Policy | Behavior |
|---|---|
| `skip` | Default. Invalid positions are excluded from decoded arrays. |
| `keep` | Invalid source rows remain in decoded arrays. This is for debugging only. |
| `error` | Decode fails if any invalid position exists. |

`ArkGaussianData` preserves traceability:

| Field | Meaning |
|---|---|
| `sourceCount` | Original source splat count. |
| `count` | Decoded output splat count after policy/limit. |
| `sourceIndices` | Source row index for every decoded splat. |
| `invalidSourceIndices` | Source rows with invalid positions. |

For the current large asset, default decode should eventually output `1,854,627` renderable splats from `1,855,266` source splats.

Regression fixture expected behavior:

```json

  "sourceCount": 4,
  "count": 3,
  "sourceIndices": [0, 2, 3],
  "invalidSourceIndices": [1]
}
```
